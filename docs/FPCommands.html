How to read command explanations

This is example command syntax:

{Parameter1}<SEP>{Parameter2}<SEP>{Parameter3}<SEP><DateTime><SEP>
Note
<SEP> - This tag must be inserted after each parameter to separate different parameters. It's value is '[]' (tab). It is the same for all commands;

Mandatory parameters:

  • Parameter1 - This parameter is mandatory, it must be filled;
  • Parameter3 - This parameter is mandatory, it must be filled;
    • A - Possible value of Parameter3;
      Answer(1) - if Parameter3 has value 'A' see Answer(1);
    • B - Possible value of Parameter3;
      Answer(2) - if Parameter3 has value 'B' see Answer(2);
  • DateTime - Date and time format: DD-MM-YY hh:mm:ss
    • DD - Day
    • MM - Month
    • YY - Year
    • hh - Hours
    • mm - Minutes
    • ss - Seconds

Optional parameters:

  • Parameter2 - This parameter is optional, it can be left blank, but separator must exist. Default: X;
    Note
    If left blank, the parameter will be used with the value after "Default:" - in this case 'X'. In some cases, a blank parameter may change the meaning of the command, which will be explained for each command;

Answer(X) - This is the default answer of the command.
Under each command there will be a list with possible answers.

Answer when command fail to execute is the same for all commands, so it will not be explained after each command.
Answer when command fail to execute:

{ErrorCode}<SEP>
  • ErrorCode - Indicates an error code;

Command: 49 (31h)

Registration of sale

Parameters of the command:

{PluName}<SEP>{TaxCode}<SEP>{Price}<SEP>{Quantity}<SEP>{DiscountType}<SEP>{DiscountValue}<SEP>{Department}<SEP>

Mandatory parameters: {PluName}, {TaxCode}, {Price}, {Department}

  • PluName - Product name ( up to 72 symbols );
  • TaxCode - Tax code (one digit 1...3);
    • '1' - VAT group A;
    • '2' - VAT group B;
    • '3' - VAT group C;
  • Price - Product price ( 0.01...9999999.99 ). With sign '-' at void operations;
  • Department - Department number (1..16);

Optional parameters: {Quantity}, {DiscountType}, {DiscountValue}

  • Quantity - Quantity of the product ( 0.001...99999.999 ). Default: 1.000;
Note
Max value of {Quantity} * {Price} is limited by a parameter (Default: 9999999.99);
  • DiscountType - Type. Default: 0;
    • '0' - no discount;
    • '1' - surcharge by percentage;
    • '2' - discount by percentage;
    • '3' - surcharge by sum;
    • '4' - discount by sum;
  • DiscountValue - Value;
    • Percentage ( 0.00...100.00 ) - for percentage operations;
    • Amount ( 0.00...9999999.99 ) - for value operations;
Note
If {DiscountType} is non zero, {DiscountValue} must contain value. If {DiscountType} is zero or empty, {DiscountValue} must be empty;

Answer:

{ErrorCode}<SEP>{SlipNumber}<SEP>{DocNumber}<SEP>
  • ErrorCode - Indicates an error code. If command passed, ErrorCode is 0;
  • SlipNumber - Current slip number - unique number of the fiscal receipt ( 1...99999999 );

Command: 53 (35h)

Payments and calculation of the total sum (TOTAL)

Parameters of the command:

{PaidMode}<SEP>{Amount}<SEP>{PaidMode2}<SEP>

Optional parameters:

  • PaidMode - Type of payment. Default: 0;
    • '0' - cash;
    • '1' - card;
    • '2' - credit;
    • '3' - tare;
      Note
      If a payment name is blank or equal to " ", using of this payment is impossible. Default payment names are: "CASH", "CARD", " ", " ".
  • PaidMode2 - Secondary payment type ( when mixed payment is necessary );
    • '0' - cash;
    • '1' - card;
    • '2' - credit;
    • '3' - tare;
      Note
      The parameter {PaidMode2} is mandatory if {Amount} is less than bill total
  • Amount - Amount to pay ( 0.00...9999999.99 ). Default: the residual sum of the receipt;

Answer:

{ErrorCode}<SEP>{Status}<SEP>{Amount}<SEP>{SlipNumber}<SEP>{DocNumber}<SEP>
  • ErrorCode - Indicates an error code;
  • Status:
    • 'D' - Returned when the paid sum is equal to the sum of the receipt. The residual sum due for payment (equal to 0) is returned to Amount;
    • 'R' - Returned when the paid sum is greater than the sum of the receipt. The change is returned to Amount;
  • Amount - The residual sum due for payment (equal to 0) or the change;
  • SlipNumber - Current slip number - unique number of the fiscal receipt ( 1...99999999 );
  • DocNumber - Global number of all documents ( 1...99999999 );
Note
If the amount of sales or returns in the receipt before the payment is equal to 0.00, the command will not be excuted. The command 'All void of a fiscal receipt' (command 60) followed by a command for closing the receipt (commands 39 or 56) should be used. After the closing of the receipt as a non fiscal, the slip number (unique number of the fiscal receipt) will not be increased;

Optional parameters:

  • Index - Used for index if variable is array. If variable is not array, "Index" must be left blank. Default: 0;
    Note

  • Value - If this parameter is blank, ECR will return current value ( Answer(2) ). If the value is set, then ECR will program this value ( Answer(1) );

Answer(1):

{ErrorCode}<SEP>
  • ErrorCode - Indicates an error code. If command passed, ErrorCode is 0;

Answer(2):

{ErrorCode}<SEP>{VarValue}<SEP>
  • ErrorCode - Indicates an error code. If command passed, ErrorCode is 0;
  • VarValue - Current value of the variable;
